## Phase 1 — Project Infrastructure & Quality Foundation

**Goal:** Establish modern development standards, quality gates, and reproducible builds.

### Tasks

1. **Add ESLint + Prettier**
   - `npm install -D eslint prettier eslint-config-prettier @typescript-eslint/parser @typescript-eslint/eslint-plugin`
   - Configure `.eslintrc.json` with TypeScript rules
   - Add `.prettierrc` with sensible defaults
   - Add lint/format scripts to package.json

2. **Add Vitest testing framework**
   - `npm install -D vitest`
   - Create `vitest.config.ts`
   - Write initial tests:
     - `src/__tests__/icons.test.ts` — validates all icon codepoints are valid hex
     - `src/__tests__/palette.test.ts` — validates color palette structure
     - `src/__tests__/css.test.ts` — validates CSS generation (no empty strings, valid syntax)
     - `src/__tests__/index.test.ts` — validates exports, init logic
   - Add `"test": "vitest run"` and `"test:watch": "vitest"` scripts

3. **Fix package-lock.json**
   - Remove `package-lock.json` from `.gitignore`
   - Commit the lock file for reproducible CI installs

4. **Add engine pinning**
   - Create `.nvmrc` with current LTS (e.g., `22`)
   - Add `"engines": { "node": ">=18" }` to package.json

5. **Add .editorconfig**
   - Consistent editor settings across contributors

6. **Add GitHub Actions CI workflow**
   - Create `.github/workflows/ci.yml`
   - Triggers: push to main, pull requests
   - Steps: checkout → setup Node → npm ci → lint → type-check → test → build
   - Use matrix testing across Node 18, 20, 22

7. **Add Dependabot config**
   - Create `.github/dependabot.yml`
   - Weekly npm dependency updates

### Deliverables
- [x] ESLint + Prettier configured and working
- [x] Vitest test suite with core coverage
- [x] GitHub Actions CI passing (lint → type-check → test → build)
- [x] `.nvmrc`, `.editorconfig`, `.gitignore` fixed
- [x] Dependabot auto-updates enabled
- [x] package-lock.json committed

## Phase 3 — SVG Framework & Tree-Shaking

**Goal:** Provide individual tree-shakeable SVG icon components alongside the font-based system.

### Tasks

1. **Generate SVG data per icon**
   - Create build script that extracts glyph data from font files
   - Output per-icon SVG files to `icons/` directory
   - Generate icon metadata (`name`, `codepoint`, `style`, `svg`)

2. **Create per-icon TypeScript exports**
   - Generate `src/svg/` directory with individual icon modules
   - Each module exports SVG path data and viewBox info
   - Add barrel export for backwards compatibility

3. **Build framework-agnostic SVG renderer**
   - `renderIcon(name, options)` function that returns SVG string
   - Support `size`, `color`, `strokeWidth`, `className` options

4. **Package restructure**
   - Add subpath exports in package.json for individual icon imports
   - Export pattern: `import { Heart } from 'abscomfonts/icons'`
   - Ensure tree-shaking works with bundlers (Vite, Webpack, esbuild)

5. **TypeScript type safety**
   - Generate `IconName` union type from icon list
   - Add typed props for SVG options

### Deliverables
- [x] Per-icon SVG data generated from font files
- [x] Individual tree-shakeable icon imports working (`import { Heart } from 'abscomfonts/svg'`)
- [x] `renderIcon()` API for dynamic SVG rendering
- [x] Full TypeScript types for all icons
- [x] Bundle size comparison vs font-only approach documented

## Phase 4 — Framework Components

**Goal:** First-class React, Vue, and Svelte components for maximum ecosystem integration.

### Tasks

1. **React package (`abscomfonts-react`)**
   - `<AbsIcon name="heart" size={24} color="red" />` component
   - TypeScript props with autocomplete for icon names
   - Tree-shakeable imports
   - Support for all modifier classes (animations, transforms, colors)
   - Bundled as separate npm package or subpath export

2. **Vue package (`abscomfonts-vue`)**
   - `<abs-icon name="heart" :size="24" color="red" />` component
   - Vue 3 composition API + TypeScript
   - Compatible with Nuxt 3

3. **Svelte package (`abscomfonts-svelte`)**
   - `<AbsIcon name="heart" size={24} color="red" />` component
   - Svelte 5 runes compatible
   - Compatible with SvelteKit

4. **Framework-specific documentation**
   - Quick-start guides for each framework
   - Props reference
   - SSR setup instructions

### Deliverables
- [x] React component package published (subpath export `abscomfonts/react`)
- [x] Vue component package published (subpath export `abscomfonts/vue`)
- [x] Svelte component package published (subpath export `abscomfonts/svelte`)
- [x] Documentation for each framework (`docs/framework-components.md`)

## Phase 5 — Icon Metadata & Discovery

**Goal:** Make icons searchable, categorized, and discoverable programmatically.

### Tasks

1. **Add icon metadata to source**
   - Create `src/icons-meta.ts` with categories, tags, keywords
   - Add style information (solid/regular/brand) per icon
   - Add aliases mapping for deprecated names
   - Add search keywords for fuzzy matching

2. **Generate search index**
   - Build script outputs `icons-index.json` with all metadata
   - Include category hierarchy
   - Include codepoint and style info

3. **Build search/filter API**
   - `searchIcons(query, filters)` function
   - Filter by category, style, tags
   - Fuzzy search across names and keywords

### Deliverables
- [x] Icon metadata file with categories and tags (`src/icons-meta.ts` + `dist/icons-index.json`)
- [x] Search index JSON generated at build time (`dist/icons-index.json`, 239 KB)
- [x] Programmatic search/filter API (`searchIcons()`, `getIconMeta()`, `getCategories()`, `getIconsByCategory()`)
- [x] Example icon picker updated to use metadata (`examples/icon-picker.html` — fetches `icons-index.json`, search + category filter)

## Phase 7 — Performance, Accessibility & Polish

**Goal:** Production-grade optimizations for real-world usage.

### Tasks

1. **CSS optimization**
   - Minify injected CSS in production
   - Option to extract CSS to external file
   - Remove unused CSS rules at build time

2. **Accessibility**
   - Auto-add `aria-hidden="true"` to icon elements
   - Support `aria-label` for semantic icons
   - Add `role="img"` and `focusable="false"` defaults
   - Screen reader testing

3. **SSR/SSG support**
   - Server-safe initialization (handle lack of `document`/`window`)
   - Hydration-safe CSS injection
   - Next.js/Nuxt/SvelteKit integration guides

4. **Performance benchmarking**
   - Measure DOM injection time
   - Measure CSS generation time
   - Benchmark rendering of 100+ icons
   - Compare bundle size vs competitors

5. **Font loading optimization**
   - Support for `font-display: swap` configuration
   - Subsetting — only load needed font weights
   - Preload strategy documentation

### Deliverables
- [x] Minified CSS in production builds (via `minifyCSS()` + tsup IIFE minification)
- [x] Proper ARIA attributes on all rendered icons (`aria-hidden`, `role="img"`, `focusable="false"`, `ariaLabel` prop on all components)
- [x] SSR-safe initialization (guarded `document`/`window` checks, string-based `fontIcon()` output, no browser globals in framework components)
- [x] Performance benchmarks documented
- [x] Font loading best practices (`font-display: optional` on all `@font-face` rules)
